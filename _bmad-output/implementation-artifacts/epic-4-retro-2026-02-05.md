# Epic 4 Retrospective: Exploration des stories

Date: 2026-02-05

## Epic Summary

- **Stories delivered:** 3/3 (100%)
- **Stories:** 4.1 Vue table, 4.2 Filtres & recherche, 4.3 Vue kanban & toggle
- **FRs couverts:** FR18, FR19, FR20, FR21, FR22, FR23

## What Went Well

- **UTable performant:** Le composable Nuxt UI `UTable` avec colonnes typées rend la table lisible et triable sans code custom. Les cellules custom (badges statut/priorité) s'intègrent bien via slots.
- **Filtres temps réel:** Les filtres (statut, priorité, epic) + recherche texte appliquent un `computed` qui filtre le tableau source — réactivité < 100ms comme requis (NFR4).
- **v-show tabs (ADR-4):** Le toggle Table/Kanban via `v-show` préserve l'état des filtres et la position de scroll entre les vues. Pas de re-render au switch.
- **StatusBadge/PriorityBadge réutilisables:** Ces composants sont utilisés dans la table, le kanban et le détail story — cohérence visuelle partout.
- **KanbanBoard 3 colonnes:** To Do / In Progress / Done avec compteurs. Les cards sont cliquables et naviguent vers `/story/[id]`. Simple et efficace pour un mode lecture.

## What Could Be Improved

- **Pas de tri persistent:** Le tri de la table (par colonne) ne persiste pas au retour depuis une story detail. Le `v-show` préserve entre table/kanban mais pas après navigation hors de la page.
- **Kanban non responsive:** Sur mobile, les 3 colonnes s'écrasent. Un scroll horizontal ou un affichage en accordéon serait meilleur.
- **EpicProgressCards click → filtre:** Le click sur une card epic filtre la table, mais il n'y a pas d'indication visuelle claire que la card est "sélectionnée" (juste le filtre qui s'applique).

## Key Insights

- Le pattern `v-show` pour les tabs est un vrai gain UX — le coût mémoire est négligeable pour un nombre raisonnable de stories (< 200), et l'expérience est fluide.
- Les auto-imports Nuxt pour `app/utils/` (getStatusColor, getPriorityIcon, etc.) simplifient beaucoup l'usage des helpers — pas de boilerplate d'import dans chaque composant.

## Action Items

1. **Kanban responsive** — scroll horizontal ou colonnes empilées sur mobile
2. **Indication visuelle filtre epic** — ajouter un état "selected" sur les EpicProgressCards (ring/border)
3. **Tri persistant** — stocker l'état de tri dans un composable partagé
